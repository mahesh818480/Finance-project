🔐 Login Page

The login page is the entry point of the application where users authenticate using their registered credentials. It ensures that only authorized users can access the dashboard and other protected routes.

The form is designed with a clean and user-friendly interface using Angular Material and Tailwind CSS. It provides real-time validation feedback to guide users while entering their credentials.

Upon successful login, the application stores user data in localStorage and redirects the user to the dashboard. This ensures session persistence even after page refresh.

If invalid credentials are entered, the system prevents access and displays appropriate error messages.

  • User must enter a valid email format
  • Password field is mandatory
  • Login allowed only for registered users
  • Invalid credentials will show error

⚠️ Login Validation

The login validation ensures that users provide correct and properly formatted inputs before submission. Angular reactive forms are used to validate inputs in real-time.

If the user enters an invalid email format or leaves required fields empty, validation messages are displayed instantly. This improves usability and prevents unnecessary server calls.

When incorrect credentials are entered, the system verifies them against stored data and blocks access.

These validations enhance security and ensure only valid users can log in.

  • Email must follow proper format ({{mockMail}})
  • Password cannot be empty
  • Error message shown for invalid login
  • Form submission blocked if invalid

📝 Register Page

The register page allows new users to create an account in the system. It collects user details such as name, email, and password.

The form uses Angular reactive forms for validation and ensures all required fields are filled correctly.

Once the user registers successfully, the data is stored in localStorage and can be used for future login.

The design focuses on simplicity and responsiveness, making it easy to use on all devices.

  • All fields are mandatory
  • Email must be unique
  • Password must be secure
  • User data stored locally

⚠️ Register Validation

The register validation ensures users provide correct and complete information before account creation.

If any required field is missing or incorrectly formatted, validation messages are shown immediately.

Password strength and confirmation validation can also be applied to enhance security.

These validations prevent invalid or duplicate user entries.

  • All fields must be filled
  • Email must be valid format
  • Password must meet criteria
  • Error messages displayed clearly

❌ Invalid Credentials

When a user enters incorrect email or password, the system identifies it and blocks access.

Instead of allowing navigation, the application displays an error message informing the user about invalid credentials.

This feature enhances security and prevents unauthorized access.

It also improves user experience by clearly indicating what went wrong.

  • Incorrect email/password rejected
  • User stays on login page
  • Error message displayed
  • No access without valid login

🔑 Forgot Password

The forgot password feature allows users to reset their password if they forget it.

Users must enter their registered email address to proceed with password reset.

The system validates the email and allows updating a new password.

This ensures account recovery while maintaining security.

  • Email must be registered
  • Validation for correct format
  • Password reset allowed
  • Error shown for invalid email

📊 Dashboard Overview

The dashboard serves as the central interface of the application, providing users with a complete overview of their financial data. It is displayed immediately after successful login and acts as the main entry point for interacting with the system.

The layout is designed using a clean and responsive structure, ensuring smooth usability across mobile, tablet, and desktop devices. It combines navigation, summary cards, and charts to deliver a seamless user experience.

Users can quickly understand their financial status through visual elements like cards and charts without needing deep analysis.

The dashboard is optimized for performance and readability, ensuring data is presented clearly and efficiently.

🧭 Navigation Bar

The navigation bar is positioned at the top of the application and provides quick access to different modules such as Dashboard, Transactions, and Insights.

It ensures smooth navigation across the application and maintains consistency in user experience.

  • Easy navigation between pages
  • Responsive design
  • Quick access to modules

💳 Cards

The dashboard displays financial data using summary cards such as Total Balance, Income, and Expense.

Each card is color-coded to help users quickly differentiate between financial metrics.

  • Displays key financial values
  • Color-coded UI
  • Quick overview

📈 Charts

The chart section visually represents income and expense data, helping users analyze trends easily.

Graphical representation simplifies complex data and improves decision-making.

  • Visual comparison of income vs expense
  • Dynamic updates
  • Easy to understand

👤 Profile Section

The profile section is available in the top-right corner and allows users to manage their account.

Users can access settings, view profile details, and securely log out from the application.
